# Content Roadmap

## Pillars
1. Personal brand and background.
2. Project proof and outcomes.
3. Skills and tools.
4. Contact and collaboration.

## Supporting Pages
1. `/` Home: positioning, quick facts, FAQ, core CTA.
2. `/about`: background, education, capabilities.
3. `/projects`: project highlights with outbound links.
4. `/skills`: skills matrix and current learning.
5. `/contact`: conversion endpoint.

## Internal Linking Plan
1. Home to Projects and Contact via explicit CTA flow.
2. About to Projects via CTA.
3. Projects to GitHub profile and future per-project URLs.
4. Footer and nav keep all key routes one click away.

## Refresh Plan
1. Update project cards monthly with project-specific links and measurable outcomes.
2. Refresh skills scores quarterly with evidence from shipped work.
3. Refresh FAQ whenever positioning, services, or contact preference changes.
4. Refresh metadata copy quarterly based on GSC CTR performance.
